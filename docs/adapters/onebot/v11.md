## ID 策略

OneBot v11 生态通常要求：

- `self_id` 是数字；
- `user_id` 是数字；
- `group_id` 是数字；
- `message_id` 是数字。

但 Oopz 的真实 ID 多数是字符串，因此 v11 adapter 使用 `IdStore` 将 Oopz 原始 ID 映射成数字 ID。

内部 source 形态大致如下：

| OneBot 字段 | source 说明 |
| --- | --- |
| `self_id` | 当前 Oopz bot 用户 UID。 |
| `user_id` | Oopz 用户 UID。 |
| `group_id` | Oopz `area + channel`。 |
| `message_id` | Oopz `area + channel + target + message_id`。 |

映射数据默认保存在：

```text
.oopz_sdk/onebot_v11.sqlite3
```

可以通过 `OneBotV11Config(db_path="...")` 指定路径。

## 消息事件

Oopz 频道消息会转为 v11 `group` 消息：

```json
{
  "post_type": "message",
  "message_type": "group",
  "sub_type": "normal",
  "group_id": 12345678,
  "user_id": 87654321,
  "message_id": 13579246,
  "message": [
    {"type": "text", "data": {"text": "hello"}}
  ]
}
```

Oopz 私聊消息会转为 v11 `private` 消息：

```json
{
  "post_type": "message",
  "message_type": "private",
  "sub_type": "friend",
  "user_id": 87654321,
  "message_id": 13579246,
  "message": [
    {"type": "text", "data": {"text": "hello"}}
  ]
}
```

## 撤回事件

频道消息撤回会转为：

```json
{
  "post_type": "notice",
  "notice_type": "group_recall",
  "group_id": 12345678,
  "user_id": 87654321,
  "operator_id": 87654321,
  "message_id": 13579246
}
```

私聊消息撤回会转为：

```json
{
  "post_type": "notice",
  "notice_type": "friend_recall",
  "user_id": 87654321,
  "message_id": 13579246
}
```

当前实现会根据 Oopz 原始事件类型、事件名，以及是否存在 `area` 来判断是频道撤回还是私聊撤回。

## 发送私聊消息

```json
{
  "action": "send_private_msg",
  "params": {
    "user_id": 87654321,
    "message": "hello"
  }
}
```

如果 `user_id` 是 adapter 已映射过的 v11 数字 ID，adapter 会自动还原为 Oopz UID。

## 发送频道消息

```json
{
  "action": "send_group_msg",
  "params": {
    "group_id": 12345678,
    "message": "hello"
  }
}
```

如果 `group_id` 未映射，需要补充 Oopz 上下文：

```json
{
  "action": "send_group_msg",
  "params": {
    "group_id": 12345678,
    "oopz_area_id": "OOPZ_AREA_ID",
    "oopz_channel_id": "OOPZ_CHANNEL_ID",
    "message": "hello"
  }
}
```

## CQ 码支持

字符串消息会解析部分 CQ 码：

| CQ 码 | 转换结果 |
| --- | --- |
| `[CQ:at,qq=123]` | `Mention(123)`，发送前会尝试还原 Oopz UID。 |
| `[CQ:at,qq=all]` | `MentionAll()`。 |
| `[CQ:image,file=file:///path/to/a.png]` | 本地图片。 |
| `[CQ:image,file=<file_key>]` | 已上传 Oopz 图片资源。 |

数组消息段也支持：

```json
[
  {"type": "text", "data": {"text": "hello "}},
  {"type": "at", "data": {"qq": 12345678}},
  {"type": "image", "data": {"file": "file:///tmp/a.png"}}
]
```

## 获取消息

`get_msg` 依赖 message mapping。只有以下消息能被查询：

- 适配器收到并保存过的事件消息；
- 适配器自己通过 `send_private_msg` / `send_group_msg` 发送成功并保存过的消息。

```json
{
  "action": "get_msg",
  "params": {
    "message_id": 13579246
  }
}
```

如果映射不存在，会返回失败响应。
